2. Preprocessing the Text

We need to clean the raw text by removing unnecessary elements from the text (such as special characters).

a) Clean Line Breaks, Whitespace, and Special Characters

Which special characters identify line breaks and extra white spaces? Use Google or ChatGPT to find out.

Then, use the gsub() function and RegEx to replace line breaks and extra spaces. Do the same to remove special characters (i.e.Â everything thatâs not a number or a letter). Hint: ^ inside a square bracket ([^...]) negates the characters inside the square brackets.

# Remove line breaks
CSS_article <- gsub("\n", " ", CSS_article)

# Remove extra whitespaces
CSS_article <- gsub("\\s+", " ", CSS_article)

# Remove special characters
CSS_article <- gsub("[^a-zA-Z0-9 ]", "", CSS_article)

b) Remove Headers and References

We donât need recurring headers (i.e., the reoccuring journal name, Journal of Computational Social Science) and the reference section for our analysis. Letâs remove those from the text by, again replacing them using gsub().

# Remove the journal header
CSS_article <- gsub("Journal of Computational Social Science", " ", CSS_article)

# Remove the references section
CSS_article <- sub("References.*", "", CSS_article) # remove after text

CSS_article <- CSS_article[1:18] # remove last two pages (only references)

c) Convert to Lowercase

R is case sensitive, i.e.Â differentiates upper and lower cases, but text analysis is typically case-insensitive. Convert the text to lowercase to ensure consistency.

# Convert all text to lowercase
CSS_article <- tolower(CSS_article)

c) Remove Frequent and Rare Tokens

Very frequent or very infrequent tokens often do not differentiate well between texts. Removing such tokens thus enables better identification of meaningful structures or frames across texts. Use quentedaâs tokens_trim() function on the token object to remove tokens that occur in either less than 1 percent of all texts or more in more than 50 percent of all texts (see ?tokens_trim()).

6. Document-Feature Matrix

A Document-Feature Matrix (DFM) is an important format for analyzing text data. It represents the frequency of terms across documents. We will now create a DFM from our cleaned, tokenized, and stemmed text data.

a) Create the DFM

Use the dfm() function from quanteda to convert the tokens object into a DFM. How many rows and how many columns does this DFM have?

# Create the DFM from tokens
dfm_CSS_article <- dfm(toks_CSS_article)
